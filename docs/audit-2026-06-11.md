## Review round 2 (adversarial, on the round-1 delta)

A second review pass over the round-1 fixes surfaced 8 findings. The auto-verifier
hit a session limit, so each was adjudicated by hand against the code and the
vendored daemon source; all 8 were real and were fixed:

- **history GC** ‚Äî the round-1 `lastSampleAt`/wall-clock anchor still had a ~60s

  `meta('last_poll')` marker is persisted in the **same transaction** as the

  ENOSPC eras, and the throttle lag. (`TestMaintainGCSafeNearOutageStart`,
  `TestMaintainGCSafeAfterWriteFailureEra`)
- **history GC** ‚Äî purge now keys on a *stale `seen` row* (`hash IN ‚Ä¶`), so a
  torrent with samples but no `seen` row is never wiped with no grace.
  (`TestMaintainGCKeepsSamplesWithoutSeenRow`)
- **history GC** ‚Äî `last_poll` is clamped to `now`, so a backward clock step
  can't make freshly-added torrents look ancient. (`TestMaintainGCSurvivesBackwardClockStep`)
- **history Sample()** ‚Äî the dedup cache and refresh throttle now advance only
  **after** `tx.Commit()` succeeds (a `pending` map applied in the deferred), so

- **gauge rollup** ‚Äî confirmed the bucket-aligned cutoff (round-1) holds under
  continuous 30s maintain passes. (`TestGaugeRollupContinuousMaintainKeepsAverage`)
- **RPC denylist (RCE)** ‚Äî the family-prefix denylist still let the **multicall
